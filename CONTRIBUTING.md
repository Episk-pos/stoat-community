# Contributing to Stoat Community

## Issues
We welcome:

- Bug reports
- Feature requests
- UX feedback
- Security reports

Please use the matching GitLab issue template so reports are triage-ready.

## Issue quality guidelines
Before submitting, ensure:

1. You searched for duplicates.
2. The issue title is specific.
3. Reproduction, expected/actual behavior, and environment are included (for bugs).
4. User problem, impact, and acceptance criteria are included (for features).
5. Screenshots, recordings, logs, or references are attached where relevant.

## Pull Requests
We accept pull requests for documentation and process improvements in this repository.
